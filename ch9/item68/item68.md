> ***일반적으로 통용되는 명명 규칙을 따르라***

<br>

### 명명 규칙

명명 규칙, 즉 네이밍 컨벤션에서 중요하게 생각하는 것은 두가지이다.

<br>

첫째, 메서드나 프로퍼티의 네이밍이 **그 동작과 속성을 최대한 명확하게 표현**해야 한다. 둘째, 다른 개발자가 보거나 시간이 지난 후 다시 봤을 때에도 그 의미가 명확하게 전달되어야 한다.

<br>

자바 플랫폼에서 제공하는 명명 규칙은 이러한 원칙들을 체계적으로 정리하여 제공하고 있고, 아래와 같은 컨벤션을 정의하고 있다.

<br>

### 철자 규칙

**패키지와 모듈**

- 계층적 구조를 사용하며 계층과 계층은 `.` 으로 구분
- 각 요소는 8자 이하의 짧은 단어 또는 약어 권장

```java
org.junit.jupiter.api
```

<br>

**클래스와 인터페이스**

- 하나 이상의 단어로 구성되며, 각 단어의 첫글자는 대문자
- 널리 통용되는 약자 외에는 사용하지 않음

```java
List, LinkedHashMap
```

<br>

**메서드와 필드**

- 첫글자는 소문자, 이후 단어의 첫글자는 대문자
- 이후는 클래스 명명 규칙과 동일

```java
remove, getCrc
```

<br>

**지역변수**

- 메서드/필드와 동일한 규칙
- 약어를 사용할 수 있지만, 문맥에서 의미가 유추되는 한에서 가능

```java
i, denom, houseNum
```

<br>

**상수 필드**

- 모든 단어를 대문자로, 단어 사이는 `_` 로 구분

```java
MIN_VALUE, NEGATIVE_INFINITY
```

<br>

**타입 매개변수**

- 보통 한 문자로 표현

```java
T = 임의의 타입
E = 컬렉션 원소
K, V = 맵의 키와 값
X = 예외
R = 메서드 반환 타입
```

<br>

### 문법 규칙

**클래스**

- 객체 생성이 가능한 경우 = 단수 명사 또는 명사구
- 객체 생성이 불가능한 경우 = 복수형 명사

```java
객체 생성 가능 = Thread, PriorityQueue
객체 생성 불가 = Collectors, Collections
```

<br>

**인터페이스**

- 클래스와 동일
- -able, -ible로 끝나는 형용사 사용 가능

```java
able = Runnable, Iterable
ible = Accessible
```
<br>


**어노테이션**

- 동사, 명사, 전치사, 형용사 등 다양하게 사용

<br>

**메서드**

- 동사 또는 동사 구 사용

```java
boolean 반환 메서드 = is 또는 has로 시작
ex) isDigit, isProbablePrime

속성 반환 메서드 = 명사 또는 get으로 시작하는 동사구
ex) size, hashCode, getTime

타입 변환 메서드 = to + Type
ex) toString, toArray

뷰 제공 메서드 = as + Type
ex) asList

기본 타입 변환 메서드 = type + Value
ex) intValue
```

<br>

**지역변수**

- 필드와 유사하지만, 더 느슨한 규칙 적용
